Grid: основные понятия

  • Grid контейнер
    Контейнер, у которого объявлено display: grid; или display: inline-grid;
    Все прямые потомки теперь являются grid элементами.
  • Grid Tracks: grid-template
    Треки - это полосы (строки/ряды и колонки/столцы) в сетке, которая состояит.
    Чтобы задать сетку и указать высоту и ширину ячекк, используем: grid-template-columns и grid-template-rows;
    или: grid-template

    Пример 1:

    grid-template-columns: 200px 200px 200px;
    grid-template-rows: 100px 100px;

    → задали грид-сетку: 3 колонки с шириной 200px каждая, а высоту указади только для 1ых двух строк - по 100px (остальные строки будут в высоту по содержимому)

    А можно задать сетку и указать значения высоты и ширины ячеек в одну строку:

    grid-template: 100px 100px / 200px 200px 200px;

    → сначала указываем строки (высоту строк), а потом ширину столбцов (таким образом, мы так же задали сетку)

  • Единица измерения fr
    Фракция (доля) - дополнительная единица измерения для размеров грид-треков. Можно миксовать с другими единицами (px, % ..).

    Пример:

    grid-template-columns: 2fr 1fr 1fr;
    grid-template-rows: 3fr 1fr;

    → ширина 1ой ячейки 2fr (2/3 от общего пространства), 2ая и 3я чейки по 1fr (1/3 от общего пространства). Доступное пространство разбивается на четыре части. Две части занимает первый трек, и две части - два оставшихся. Высота 1ой строки занимает 3 части от общей высоты, 2ая тсрока занимает 1/3 т общей высоты

    Пример:

    grid-template: 3fr 1fr / 500px 1fr 1fr;

    → Высота 1ой строрки - 3 доли от обещй высоты, 2ой строки - 1 доля. Ширина первой колонки 500 пикселей, и эта фиксированная ширина убирается из доступного пространства. Оставшееся пространство разбивается на три части и пропорционально разделяется между двумя колонки.

  • Явный (grid-template-*) и неявный грид (grid-auto-*)
    Когда элементы расположены за пределами сетки (например, элементов больше, чем в заданном правиле), Grid-контейнер генерирует неявные полосы путем их добавления в общий Grid (высота ячеек будет автоматической - по высоте содержимого).
    grid-template-* применяется для создания явного Grid: grid-template-rows, grid-template-columns, grid-template-areas
    Свойства grid-auto-* чтобы определить размер неявного Grid, то есть высоту всех элементов, для которых правило не указано явно:
    grid-auto-rows, grid-auto-columns или grid-auto-flow
    grid-template-rows: 100px 100px;

    → у первых двух строк высота будет по 100px, а у остальных высота будет определятся содержимым.

    Пример 1:

    grid-auto-rows: 30px;

    → определили значение высоты, и этот стиль будет применен ко всем строкам в Grid (для которых не яказана явно высота через grid-template).

    Пример 2:

    grid-ayto-rows: minmax(100px, auto);

    → для строк с явно не заданной высотой - указали, что высота будет минимум 100px и максимум - автоматически (по содержимому контента)

  • Зазоры Gutters (отступы)
    grid-column-gap, grid-row-gapd → с их помощью можно задавать отступы (зазоры) между элементами сетки

    Пример 1:

    grid-gap: 15px;

    → Задали отступ между ячейками - 15px

    Пример 2:

    grid-column-gap: 10px;
    grid-row-gap: 1em;

    → Задали отступ между колонками - 10px и отступы между строками - 1em

  • Задание треков с помощью нотации repeat ()
    Функция repeat() принимает два значения: количество повторений и значение, которое надо повторить (обычно, для большого количества треков).

    Пример 1:

    grid-template-columns: 1fr 1fr 1fr;

    А можно переписать так:

    grid-template-columns: repeat(3, 1fr);

    Пример 2:

    grid-template-columns: 20px repeat(6, 1fr) 20px;

    → создается грид с начальным треком (колонкой) шириной в 20px, шестью треками шириной в 1fr и последним треком шириной в 20px

    Пример 3:

    grid-template-columns: repeat(5, 1fr 2fr 200px);

    → грид состоит из 10 треков (колонок): за треком шириной 1fr следует трек шириной 2fr потом трек в 200px, и весь этот набор повторяется пять раз

  • Масштабирование треков: minmax(), max-content и min-content
    minmax(min, max)- функция для треков, которая задает и минимальное, и максимальное значения.

    Пример 1:

    → grid-template-columns: minmax(100px, 400px) 1fr 1fr;

    Первый трек (колонка) остается в пределах от 100px до 400px. При изменении размеров окна абсолютное значение будет меняться, но всегда между этими границами. Вторая и третья колонки сжимаются и расширяются до заполнения оставшегося свободного места поровну, но первая всегда остается шириной от 100px до 400px.

    Пример 2:

    grid-template-columns: minmax(200px, 1fr) 1fr 1fr;

    → трек (колонка) должна быть шириной минимум 200px. Если окно браузера станет шире этого, эта колонка должна быть размером 1fr, т.е. равна двум другим колонкам

    Пример 3:

    grid-ayto-rows: minmax(100px, auto);

    → для строк с явно не заданной высотой - указали, что высота будет минимум 100px и максимум - автоматически (по содержимому контента)

    Пример 4:

    grid-template-columns: max-content 1fr 1fr

    → первая колонка растягивается в ширину по содержимому контента, чтобы вместить всю длину строки

    Пример 5:

    grid-template-columns: min-content 1fr 1fr

    → содержимое первой ячейки переносится так, чтобы занимать как можно меньше места по горизонтали, не вызывая переполнения.

    Пример 6:

    grid-template-columns: minmax(auto, auto) 1fr 1fr;

    → если значение auto используется в качестве max, оно эквивалентно значению max-content. А если в качестве min, то значение auto представляет собой наибольший минимальный размер для ячейки

    grid minmax
    → хорошее объяснение + примеры

  • для создания адаптива можно использовать - auto-fill, auto-fit
    auto-fill- можно использовать с функцией repeat() вместо количества повторений. Оно гибко меняет используемое число колонок в зависимости от ширины каждой из них. Применяется для создания адаптива.

    Пример :

    grid-template-columns: repeat(auto-fill, minmax(200px, 1fr));

    → у каждой колонки в гриде минимальная ширина 200px. При изменении размеров окна браузера число колонок меняется, чтобы вместить их идеальную ширину.
    auto-fill - когда общего пространства для ячеек станет мало (а у каждой ячейки мин. ширина - 200px), то ячейки будут переноситься на новую строку

    auto-fit- тоже с функцией repeat() вместо количества повторений. В отличае от auto-fill - при переносе ячеек на новую строку они будут прижиматься не к левому краю, а равномерно распределяться по контейнеру.

    Пример :

    grid-template-columns: repeat(auto-fit, 150px);     При этом, медиа- запросы @media- никто не отменял)). То есть, к примеру, с помощью медиа-запросов, мы можем изменить сетку грид-контейнера, а так же переписать положение и области для блоков grid-template-areas

    Пример :

    @media all and (min-width: 0) and (max-width: 767px) {
    .grid {
    grid-template-columns: 1fr; grid-gap: 0;
    grid-template-areas:
    "header"
    "nav"
    "main"
    "aside"
    "voting"
    "footer"
    } }
  • Выравнивание содержимого сетки justify-items (вдоль оси строки)
    justify-items Выравнивает содержимое вдоль оси строки. Это значение применяется ко всем элементам сетки внутри контейнера.
    Принимаемые значения: start, end, center, stretch (по-умолчанию)

    Пример 1:

    justify-items: start;

    → ввыравнивает содержимое сетки по левой стороне.

    Пример 2:

    justify-items: stretch;

    → растягивает ввыравнивает содержимое сетки (значение по-умолчанию).

  • Выравнивание содержимого сетки align-items (вдоль оси столбца)
    align-items Выравнивает содержимое вдоль оси столбца. Это значение применяется ко всем элементам сетки внутри контейнера.
    Принимаемые значения: start, end, center, stretch (по-умолчанию)

    Пример:

    align-items: start;

    → выравнивание элементов по верхней части области

  • Выравнивание сетки justify-content (оси строки)
    justify-content Выравнивает сетку вдоль оси строки
    Принимаемые значения: start, end, center, stretch (по-умолчанию), space-around, space-between, space-evenly

    Пример:

    justify-content: stretch;

    → масштабирует элементы чтобы сетка могла заполнить всю ширину контейнера.

    justify-content: space-around;

    → одинаковое пространство между элементами, и полуразмерные отступы по краям.

    justify-content: space-between;

    → одинаковое пространство между элементами, без отступов по краям.

    justify-content: space-evenly;

    → одинаковое пространство между элементами, и полноразмерные отступы по краям.

  • Выравнивание сетки align-content (оси колонки)
    align-content выравнивает сетку вдоль оси колонки
    Принимаемые значения: start, end, center, stretch (по-умолчанию), space-around, space-between, space-evenly

    Пример:

    align-content: end;

    → выравнивает сетку по нижней части контейнера.

    Пример:

    align-content: center;

    → масштабирует элементы чтобы сетка могла заполнить всю высоту контейнера.

  • Выравнивание содержимого элемента justify-self (вдоль оси строки)
    justify-self выравнивает содержимое элемента вдоль оси колонки
    Принимаемые значения: start, end, center, stretch (по-умолчанию)

    Пример:

    div:nth-child(9) { justify-self: start; }

    → выравниваем содержимое 9ого элемента по левой части области.

  • Выравнивание содержимого элемента align-self (вдоль оси столбца)
    align-self выравнивает содержимое элемента вдоль оси колонки
    Принимаемые значения: start, end, center, stretch (по-умолчанию)

    Пример:

    div:nth-child(5) { align-self: end; }

    → выравниваем содержимое 5ого элемента по нижней части области.

  • Положение, область и порядок элементов: grid-*-start, grid-*-end и grid-auto-flow
    При размещении элемента мы ссылаемся именно на линию, а не на трек (колонку/строку).
    grid-column-start, grid-column-end, grid-row-start, grid-row-end → с их помощью можно размещать элементы в любом положении в грид-таблице

    Пример 1:

    .wrapper { display: grid; grid-template-columns: repeat(3, 1fr); }
    div:nth-child(1) {
    grid-column-start: 2;
    grid-row-start: 3;
    }

    → Задали сетку - в 3 колокни. Выбрали первый дочерний элемент грид-контейнера и задали ему положение: 2ая колонка, 3я строка

    Пример 2:

    .wrapper { display: grid; grid-template-columns: repeat(3, 1fr); }
    .box1 {
    grid-column-start: 1; grid-column-end: 4;
    grid-row-start: 1; grid-row-end: 3;
    }

    Или сокращенно:

    .box1 {
    grid-column: 1/4;
    grid-row: 1/3;
    }

    → Задали положение и область для блока box1. Он занимает область начиная с 1ой колоночной линии, и занимает пространство до 4ой колоночной линии (т.е. 3 элемента в строке). В высоту область начинается со строчной линии 1 и заканчивается на строчной линии 3, (т.е. 2 элемента в высоту). Остальные элементы самостоятельно размещаются в пустых пространствах грида

    Поменять последовательности элементов можно с помощью grid-auto-flow принимает значения: row, column, dense

    Пример:

    grid-auto-flow: column;

    → последовательность блоков будет выстраиваться не по строкам, а по колонке

  • Обратиться к ячейке или к треку (колонке/столбцу)
    Обратиться к определенной ячейки или к треку (колонке/столбцу) - можно через всевдоклассы.

    Пример:

    .container {display: grid; grid-template: 1fr 1fr / 1fr 1fr 1fr}
    div:nth-child(1);

    → обратились к 1ой ячейке. Где div - это дочерний элемент grid-контейнера

    Пример:

    .container {display: grid; grid-template: 1fr 1fr / 1fr 1fr 1fr}
    div:nth-child(3n-1);

    → обращаемся к каждому 2ому элементу. Так как наша сетка имеет 3 колонки, мы обращаемся ко всей 2ой колонке;

  • Выстроить шаблон grid-template-areas
    grid-area Можно выстроить шаблон grid-template-areas с помощью псевдонимов grid-area. сначала для блоков задаем имена с помощью grid-area. Потом, в самом грид- контейнере (где расположены эти блоки), можем выстроить шаблок, как будут располагаться блоки, в какой позиции, сколько треков занимать

    Пример 1:

    header { grid-area: header;}
    .nav { grid-area: nav;}
    ...
    .grid {
    display: grid;
    grid-template-colimen: repeat(3, 1fr);
    grid-template-areas:
    "header header header"
    "nav main aside"
    ". main voting"
    "footer footer footer"
    }

    → Задали блоку header псевдоним - "header" и блоку .nav - псевдоним "navbar" и так далее для других блоков.
    Далее, для грид-контейнера можем выстроить шаблок из псевдонимов (они буду располагаться так, как мы напишем). "." - означает пропуск.

  • Прижать footer grid

    Пример 1:

    html, body {
    height: 100%; margin: 0;
    }
    body {
    display: grid;
    grid-template-rows: 200px minmax(500px, 1fr) 200px;
    }

    → footer всегда будет прижат к низу (для него даже не нужно что-либо еще прописывать)
